# 博士健身日志

一个给自己使用的轻量训练记录小工具，已经部署在 GitHub Pages：

[https://wqs1021.github.io/personal-workout-log/](https://wqs1021.github.io/personal-workout-log/)

## 这个项目解决什么

这个小程序用来记录每次训练的动作、重量、次数、有氧数据，并按日期保存在当前浏览器本地。它不依赖账号、不需要云同步，也不会把训练数据上传到服务器。

适合这样的使用方式：

- 每天训练时打开手机网页，按当天计划逐项填写。
- 有氧运动可以记录椭圆机、爬楼机、跑步机爬坡、单车等不同方式。
- 动作做腻了或器械变化时，可以在“计划”里直接改动作。
- 训练数据按日期保存，后续可以在“统计”里查看本周、月度和动作趋势。

## 主要功能

- 训练记录：记录重量、次数、时长、心率、热量等数据。
- 按日期保存：每天的训练内容独立存储，方便回看。
- 本地缓存：数据保存在浏览器 `localStorage`，不需要云端数据库。
- 训练计划编辑：可以添加、修改、删除每周训练动作。
- 动作库：内置常见器械、哑铃、自重、有氧动作，添加计划时可以直接套用。
- 计划复制：可以把当前日计划复制到其他日期，也可以从其他日期复制过来。
- 有氧运动记录：支持选择有氧方式，例如椭圆机、爬楼机、跑步机爬坡、慢跑、单车。
- 上次记录提示：每个动作会显示上一次填写的数据，训练时更容易接上进度。
- 数据统计：显示本周完成率、月度概览、训练天数、力量组数、有氧时长、有氧热量、最近重量和动作趋势图。
- 数据导出：支持导出 Excel，方便查看和留档。
- JSON 备份/恢复：用于迁移浏览器或避免误清数据。
- PWA 支持：添加到手机桌面后更接近 App 使用体验，并带基础离线缓存。
- 历史数据保护：动作使用稳定 ID 并在记录中保存快照，修改计划不会让旧记录错位。

## 数据隐私

训练数据只存储在你当前使用的浏览器里。

需要注意：

- 换手机、换浏览器或清理浏览器数据后，本地记录可能丢失。
- 建议定期使用“备份 JSON”保存一份本地备份。
- GitHub Pages 只托管网页代码，不保存你的训练数据。

## 使用建议

1. 打开线上地址。
2. 在手机浏览器里选择“添加到主屏幕”。
3. 每次训练前进入“计划”，确认当天动作是否合适。
4. 训练时点击动作卡片填写数据。
5. 每周查看“统计”，观察执行情况、月度累计和动作趋势。
6. 定期导出 JSON 备份，必要时再导出 Excel 留档。

## 本地运行

这个项目是纯静态网页，不需要安装依赖。

直接打开 `index.html` 即可使用。也可以在项目目录启动一个本地静态服务：

```bash
python3 -m http.server 8765
```

然后访问：

```text
http://127.0.0.1:8765/
```
